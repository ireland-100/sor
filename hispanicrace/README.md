# Hispanic Race Identification in the 2020 U.S. Census

This repository analyzes the top three race categories that the Hispanic population selects on the 2020 Census, and asks which demographic, economic, and political factors predict those choices. The three categories studied throughout are:

1. **Some Other Race alone** (SOR)
2. **White + Some Other Race** together
3. **White alone**

The analysis combines 2020 Decennial Census (PL) data, 2020 American Community Survey 5-year estimates, TIGER/Line geographies, and county/tract-level 2020 presidential election results. Methods include descriptive correlations, spatial-autocorrelation statistics (Moran's I and the CAPY half-edge score), dual-graph visualization, logistic regression, and machine-learning models (random forests with SHAP, plus regularized linear baselines).

## Repository contents

### Notebooks

| Notebook | Geography | What it does |
|---|---|---|
| `3hispanicclean.ipynb` | National, county-level | Ingests and merges all data sources, computes race shares and ratios, runs scatterplots/correlations, spatial analyses (Moran's I + half-edge), a dual-graph map, and univariate logistic regressions. **Exports `nation_census_and_geodata.parquet`**, which the prediction notebook consumes. |
| `statetractcleaned.ipynb` | Single state, tract-level | The same descriptive and spatial pipeline as the national notebook, but at Census-tract resolution for one state at a time (California, Texas, Florida, or New York). |
| `nationhispanicraceprediction.ipynb` | National, county-level | Predictive modeling on the exported parquet: dominant-race classification (baseline / L1-logistic / random forest), continuous-proportion regression with SHAP, and several alternative regressors. |

### Data files

You need to provide these CSVs (they live alongside the notebooks):

- **`county_level_2020.csv`** — county-level 2020 presidential election results. Read by `3hispanicclean.ipynb`.
- **`CA_tract_election_and_pop_data.csv`** (and the equivalent file for any other state you run) — tract-level election and population data. Read by `statetractcleaned.ipynb`. The election data was sourced from <https://github.com/dra2020/block_data> and grouped to the tract level. California, Texas, Florida, and New York have prepared CSVs; for any other state you must build the CSV yourself.

The following are **not** stored in the repo because they are fetched or generated automatically:

- **Census API data** — pulled live from the Census Data API at runtime (requires an API key; see below).
- **TIGER/Line geographies** — downloaded directly from `www2.census.gov` at runtime via `geopandas.read_file(url)`.
- **`ca_dual_graph_a.json`** — a temporary dual-graph file written and then re-read within a single cell.
- **`nation_census_and_geodata.parquet`** — generated by `3hispanicclean.ipynb` and consumed by the prediction notebook. Run the national notebook first to produce it. (You may optionally commit this file if you want the prediction notebook to run standalone.)

## Census API key (required)

As of **May 12, 2026**, the U.S. Census Bureau requires an API key for **all** Census Data API queries. The two data-fetching notebooks read the key from an environment variable, falling back to a hard-coded value if the variable is not set:

```python
import os
CENSUS_API_KEY = os.environ.get("CENSUS_API_KEY", "your_key_here")
```

Request a free key at <https://api.census.gov/data/key_signup.html>, then set it in your environment before launching Jupyter:

```bash
export CENSUS_API_KEY="your_key_here"      # macOS / Linux
# setx CENSUS_API_KEY "your_key_here"       # Windows (new shell after running)
```

**Do not commit a real key.** If you keep the hard-coded fallback while developing, delete it before pushing. The prediction notebook needs no key, since it only reads the local parquet file.

## How to run

1. Clone the repo and install the dependencies (see below).
2. Request a Census API key and set `CENSUS_API_KEY` in your environment.
3. Make sure the required CSV(s) are in the same directory as the notebooks.
4. Run `3hispanicclean.ipynb` top to bottom. This fetches the data, produces the national analysis, and writes `nation_census_and_geodata.parquet`.
5. Run `nationhispanicraceprediction.ipynb` to reproduce the predictive models (it reads the parquet from step 4).
6. To run the tract-level analysis, open `statetractcleaned.ipynb`. To switch states, update the state FIPS code in three places: the `in='state:NN'` parameter in `get_census_data`, the TIGER tract-geometry URL, and the election-CSV filename. Make sure the matching state CSV is present.

## Dependencies

The notebooks use:

- `pandas`, `numpy`
- `geopandas`, `libpysal`, `esda`, `gerrychain` (spatial data and statistics)
- `scipy`, `statsmodels` (statistics / logistic regression)
- `scikit-learn` (predictive models)
- `shap` (model interpretation)
- `matplotlib`, `networkx`, `seaborn` (visualization)
- `requests` (Census API calls)
- `pyarrow` or `fastparquet` (parquet I/O)

Install with:

```bash
pip install pandas numpy geopandas libpysal esda gerrychain scipy statsmodels scikit-learn shap matplotlib networkx seaborn requests pyarrow
```

A virtual environment (e.g. `python -m venv .venv`) is recommended.

## Notes

- Notebook outputs (printed metrics, tables, plots) are saved in the `.ipynb` files. Re-running regenerates them and may overwrite the committed versions.
- The relative file paths assume the CSVs sit in the same folder as the notebooks. If you store data in a subfolder, update the paths accordingly.
- Run the national notebook before the prediction notebook, since the latter depends on the parquet the former produces.
